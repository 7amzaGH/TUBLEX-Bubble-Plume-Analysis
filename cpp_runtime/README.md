# TUBLEX C++ Embedded Runtime Skeleton

This folder contains a lightweight C++ runtime prototype for the **TUBLEX Bubble Plume Analysis** repository.

The goal is to demonstrate how the structured feature-level monitoring stage can be expressed in an embedded-friendly C++ structure.

This component does **not** perform video processing, bubble detection, OpenCV preprocessing, DBSCAN filtering, Random Forest `joblib` loading, XGBoost inference, SHAP explainability, model training, camera integration, underwater robot control, SLAM, or full onboard AUV deployment.

It is intentionally limited to the **feature-level runtime stage** after TUBLEX one-second window features have been extracted.

---

## Purpose

The runtime reads TUBLEX window-level feature rows exported from the Python pipeline and applies a lightweight embedded-style suspicion scoring interface:

```text
TUBLEX one-second feature CSV
        ↓
C++ feature-window loader
        ↓
Embedded-style suspicion scoring interface
        ↓
NORMAL / MONITORING / SUSPICIOUS state
```

This is useful for showing how the lightweight structured-feature part of TUBLEX can be moved from Python experimentation toward an embedded-style C++ implementation.

---

## Runtime Outputs

| Output | Meaning |
|---|---|
| `window_id` | One-second TUBLEX window index |
| `suspicion_score` | Lightweight runtime suspicion score in `[0, 1]` |
| `state` | Runtime state: `NORMAL`, `MONITORING`, or `SUSPICIOUS` |
| `mean_count` | Mean bubble count in the current window |
| `std_count` | Bubble-count variability in the current window |
| `continuity` | Ratio of frames containing detected bubbles |
| `vertical_chain` | Mean vertical bubble-chain descriptor |

## Build

From inside `cpp_runtime/`:

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

---

## Run

Run the included mixed example:

```bash
./tublex_embedded_runtime ../examples/sample_mixed_windows.csv
```

Run with custom suspicion and monitoring thresholds:

```bash
./tublex_embedded_runtime ../examples/sample_mixed_windows.csv 0.60 0.35
```

On Windows with Visual Studio generators, the executable may be inside the build configuration folder:

```bash
./Debug/tublex_embedded_runtime.exe ../examples/sample_mixed_windows.csv
```

---

## Input Format

The runtime reads a CSV file containing one row per one-second TUBLEX window.

Required columns:

```csv
window_id,mean_bubble_count,max_bubble_count,std_bubble_count,continuity_ratio,mean_vertical_chain,temporal_variance,prev3_mean_bubble_count,prev3_std_bubble_count,prev3_continuity_ratio,prev3_mean_vertical_chain,leak_evolution
```

Example row:

```csv
0,0.4,2,0.7,0.2,0.3,0.5,0.0,0.0,0.0,0.0,0.4
```

The example CSV files are intentionally small and human-readable. In the full Python pipeline, these rows are generated by the TUBLEX video processing and temporal feature extraction modules.

## Scope Note

This C++ runtime is an **embedded-oriented extension**, not the main experimental contribution of the repository.

It does not claim:

- real-time onboard AUV deployment,
- closed-loop control,
- SLAM or underwater localization,
- raw video bubble detection in C++,
- direct Random Forest or XGBoost model reproduction,
- SHAP explainability in C++,
- physical underwater deployment validation.

Its purpose is to show how TUBLEX feature-level monitoring can be represented as a clean C++ component suitable for future embedded integration.

---

## Relation to the Python Pipeline

The Python pipeline is used for preprocessing, bubble detection, temporal feature extraction, training, evaluation, visualization, SHAP analysis, and model comparison.

The C++ runtime focuses only on the lightweight feature-level monitoring stage.

```text
Python TUBLEX pipeline
        ↓
Video processing, feature extraction, ML evaluation, SHAP analysis

C++ runtime skeleton
        ↓
Embedded-style feature-window monitoring prototype
```
